# 「另一个你 · Other You」

> 占位名，正式名待定。

把一个真实的决定丢给 Claude Code，每个选项长成一条 90 天的时间线，在单文件 HTML 的像素小世界里，并排看几个「你」各走一段日子。

不是人生模拟，不是命运预测。是把「如果当初选了 B」这句话，变成一段可以拖动进度条看的、写在你真实处境上的具体故事。

## 3 步用法

1. 在这个目录打开 Claude Code，说一句类似的话：「按 PIPELINE.md 跑一个决定：选项 = 接 offer / 读研 / 创业，跨度 = 90，档案 = 真实自我」
2. Claude Code 照着 `PIPELINE.md` 的管线，产出一份新的推演文件并注册进选择页
3. 双击 `index.html`，打开是选档案页（真实自我 / 虚拟人设 / 名人档案三档），点开刚生成的那份看结果；历史推演都留在选择页里，不互相覆盖

零后端、零 API、零安装。整条链路就是 **Claude Code 推演 → data.js → index.html**。

## 数据档案三档

生成每条时间线时，Claude Code 需要知道「你是谁」。档案分三档，隐私强度不同：

| 档位 | 内容 | 存放位置 | 会不会进 git |
|---|---|---|---|
| 🔒 真实自我（real） | 从你自己的笔记手工蒸馏出的处境摘要 | 档案在 `profiles/real/`，推演产物在 `runs/real/` | 都不会，两处都被 `.gitignore` 挡住；选择页的这一档只在你本机有内容 |
| 🎭 虚拟人设（persona） | 编造的一个人，用于演示/测试 | `profiles/persona-example.md` | 会，本来就是模板 |
| 🌟 名人公开档案（celebrity） | 公开资料现场调研，不需要本地档案文件 | 无需文件 | 会（本来就是公开信息） |

## 隐私铁律

- 真实自我档案（`profiles/real/*`）**永不出本机**，`.gitignore` 已经挡死，不会被 commit，也不会被推到任何仓库。
- 推演过程不调用外部 API、不入库、不上传。所有计算发生在你本机的 Claude Code 会话里，产物只是一个本地 `data.js` 文件。
- 真实档案生成的每条时间线，在正式使用前建议本人过目摘要是否失真（详见 `PIPELINE.md`）。

## 免责声明

这是**可能性的戏剧化演出，不是预测**，更不是人生建议。每条时间线只是「在这个设定下，可能发生的一种走向」，不代表真会发生，也不构成任何决策依据。

名人相关内容（celebrity 档）一律标注为**虚构推演 · 娱乐向**：在世人物不做负面杜撰，虚构走向只按「合理但娱乐向」处理。

## 技术形态

- 单文件 HTML，Canvas 渲染的像素风小世界
- 零依赖、零后端、零 API，`file://` 双击即可运行
- 数据层与展示层分离：每个决定是 `runs/` 下一个独立文件（契约见 `SCHEMA.md`），打开网页在选择页里挑

## 和市面 demo 的区别

市面上不少「人生模拟器」「平行人生」类的 demo，玩的是通用随机事件，换个名字套谁都一样。这个项目反过来：**先有你的真实处境，再有时间线**。每一条 `reasoning` 都必须扣着档案里写的具体处境（资源、软肋、纠结）来解释「为什么会发生」，不是抽卡式的随机事件池。

v0.5 计划加入「现实校准」：随着时间推移，把你实际发生的事回填进去，让时间线跟着现实改写，而不是生成一次就定型。

# 推演管线操作手册

这是「另一个你 · Other You」的引擎层：如何让 Claude Code 把一个决定变成一份 `data.js`。

## 使用方式

在这个目录（`/Users/max/code/parallel-life/`）打开 Claude Code，直接说一句类似这样的话：

> 按 PIPELINE.md 跑一个决定：选项 = 接 offer / 读研 / 创业，跨度 = 90，档案 = 真实自我

Claude Code 会读这份文档、读 `SCHEMA.md`、读对应档案文件，然后生成一份新的 `data.js`。

几个可调参数：

- **跨度（spanDays）**：可选 30 / 90 / 180，不说默认 90。
- **选项数**：不限，但 2~4 条最佳（多于 4 条画面会挤，少于 2 条没有对比意义）。
- **档案（profileTier）**：`real` / `persona` / `celebrity` 三选一，选法见下方「三档档案怎么选」。

跑完之后，双击根目录的 `index.html` 就能看到结果。想换一个决定重跑，覆盖 `data.js` 即可，`data.sample.js` 是只读参考不要动。

## 生成提示词模板

把下面这段发给 Claude Code（方括号里填具体内容），或者直接用「使用方式」里那句口令，Claude Code 应该能自己展开成这个模板：

```
请按 SCHEMA.md 的数据契约，为下面这个决定生成 data.js。

决定：[一句话描述要做的决定]
选项（2~4 个）：
1. [选项 A 短语，≤15 字]
2. [选项 B 短语，≤15 字]
3. [选项 C 短语，可选]

跨度：[30 / 90 / 180] 天
档案档位：[real / persona / celebrity]
档案来源：[real → profiles/real/ 下的具体文件；persona → profiles/persona-example.md 或指定文件；celebrity → 说明是谁 + 现场调研公开资料]

严格要求：
- 输出必须是合法的 window.SANDBOX_DATA = {...}，字段和类型完全按 SCHEMA.md 来，不要自创字段。
- 每条时间线（timeline）15~25 个事件，按 day 升序，覆盖整个跨度，不要前松后紧或前紧后松地堆在一头。
- 每个事件的 reasoning 必须引用档案里写到的真实处境（资源、软肋、当前纠结等具体细节），不能是套话。如果档案里没有足够细节支撑某个事件，换一个能被档案支撑的事件，而不是编空话。
- 四类事件（机会 / 挫折 / 意外 / 转折）在每条线里都要出现，不要只堆机会或只堆挫折。
- effects 要让状态面板真的动起来：每条线全程，四个维度（energy / money / skill / options）各自至少变化 3 次。
- 每条线写 finale（gave / took / surprise）三句话收尾。
- 最后生成一个全局的 optionC：一条谁都没选、但值得想一想的路。
- meta.disclaimer 固定写「这是可能性的戏剧化演出，不是预测」；如果是 celebrity 档，meta.tags 必须包含「虚构推演 · 娱乐向」。
- 措辞通篇用「可能」「或许」，不写成预测、不写成建议。
```

## 质量红线

生成任何数据前后，对照检查这几条（详见 `SCHEMA.md` 末尾，这里展开说明）：

1. **绝不卖预测。** 每条时间线是「在这个设定下可能发生的一种走向」，不是「你如果这样选就会怎样」。全篇措辞只能是「可能」「或许」，不能出现「一定会」「你应该」这类断言或建议式表达。
2. **事件不能是空话。** `reasoning` 必须能指回档案里写的具体处境（比如档案提到"存款只够撑两个月"，事件才能因此触发"存款告急"）。如果一个事件套在谁身上都成立（星座运势式），说明它没有扣住档案，需要重写。
3. **celebrity 档案的两条红线：**
   - `meta.tags` 必须包含「虚构推演 · 娱乐向」。
   - 在世人物**不做负面杜撰**：不编造损害名誉、违法、丑闻类的虚构情节。已故人物或历史事件相对宽松，但仍要「合理」而不是恶搞。
4. **真实档案生成前，Max 要先过目摘要。** 用 `real` 档案跑管线时，Claude Code 应该先把从 `profiles/real/` 读到的处境摘要（几句话概括：处境、资源、软肋、当前纠结）说给 Max 看，确认没有失真或误读之后，再往下生成完整的 data.js。这一步不能省，因为真实档案直接决定了每条线的 reasoning 是否站得住。

## 三档档案怎么选

- **real（真实自我）**：档案文件放在 `profiles/real/` 下，是从 Max 自己的笔记手工蒸馏出来的处境摘要（不是原始笔记本身）。这个目录被 `.gitignore` 保护，不会进 git。用这一档时，Claude Code 先读文件内容确认摘要，过一遍「质量红线」第 4 条再生成。适合真正要做决定的场景。

- **persona（虚拟人设）**：以 `profiles/persona-example.md` 为模板，可以直接用这个示例人设，也可以照着格式另写一份新的虚拗人设文件。适合演示、测试、给别人看效果，不涉及任何真实隐私。

- **celebrity（名人公开档案）**：不需要本地档案文件，Claude Code 现场调研这个人在决定发生那个时间点的公开资料（生平、处境、当时的选择及后续影响），作为生成依据。这一档默认自带回测视角：如果历史上这个人真的做了某个选择，对应那条 timeline 的 `isRealHistory` 设为 `true`，并在相关事件上加 `realityCheck` 字段做「史实对照」。
